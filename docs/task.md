Entrance Exam:

Objective:
Developing a backtesting engine capable of replaying historical data and evaluating strategy performance

Programming languages: C++ or Python

Minimum Requirements
- Limit order book simulation
- Limit order placement and cancellation
- Order execution modeling
- Partial fills optional

Basic metrics: PnL, inventory, turnover

Strategy Implementation
- Implement Avellaneda–Stoikov (2008)
- Enhance using microprice + Avellaneda–Stoikov (2018) extensions
- Run simulation experiments

Execution Assumption
Execution occurs when the market price crosses the order level

Deliverables

Backtester:
- Integrated backtest engine
- Sample dataset and configs
- Performance report
- Technical documentation

Strategy:
- Source code
- Model description
- Performance results
- Improvement roadmap
